Gardening App

This was a final group project with four team members. The idea was to be able to build a virtual garden that represents your actual garden at home. The initial goal was to connect the application to plant/garden APIs for tips and tricks about growing better. We found that there were no good plant APIs so we seeded our Mongo database with data that we scraped off of a popular plant website. If a user cant find a plant among the list of popular plants we created, they could add their own. For the rarest of plants users could upload an image to our applicaiton using AWS as an image hosting service. We implimented this using AWS-S3 buckets. Lastly, each plant and garden has a notes section where users could comment on gardens and plants. Version control and agile/scrum were also part of the workflow for this project. My responsibilities were too impliment CRUD Functionality for users, gardens and plants.

TECHNOLOGIES USED:

  • MERN Stack
  • Full Stack CRUD (MERN)
  • React (with hooks)
  • weather API
  • Backend User Auth (private components were never implimented)
  • AWS - S3

NYT Article Scraper

Full Stack web scraping application. This project was meant to build skills in breaking down the DOM Tree of a well known website (in this case The New York Times) and scrape useful data off the website using various query selectors. After the front page of the website is scraped (in this case for the first 30 articles) a user can delete the articles they dont like and make notes on articles they find interesting. Notes and artiles are stored in an MongoDB back end. Cheerio npm packge was used as a jQuery mimicing query selector tool. Morgan npm package was used to log requests.

TECHNOLOGIES USED:

  • JQuery
  • Handlebars
  • MongoDB (mongoose)
  • Axios
  • Bootstrap
  • Cheerio

Friend Finder

Tinder for friends. This app has an express backend that renders two different files. No view engine is used. instead the app utilizes the node path dependency to send two different html pages (survey.html & home.html) upon request. The flow is such that the user will answer some questions on the survey.html view. when the form data is submitted, answers are compared with a dataset of other responses. the other responses are fabricated in a js file (friends.js) that sits in a data directory. The made up person who's answers are closest to yours will be your friend match.

TECHNOLOGIES USED:

  • Express
  • Path

Star wars game

Each star wars character is a JS object with a life-force property and hit-power property. You begin by selecting your player and your opponent. When you attack your opponent your chance of hitting is randomized. If you hit you damage your opponents life force. Your damage ability is affected by your hit-power. If you defeat your opponent you absorb some of their life-force making you stronger. You win the game by defeating all opponents. JavaScript template strings are used to display useful information in the game message-box.

TECHNOLOGIES USED:

  • JavaScript
  • OOP

LIRI BOT

Command line entertainment application where a user can search artist/song information via command line arguements. I.e. a user can search "i want it that way" which will call the spotify api and display artist name(s), and album name -> Backstreet Boys-Millenium . A user can search for performances of an artist they like by calling the bands in town api which will resond with a list of cities, venues and dates for an artist. Users can call the OMDB api with a name of a movie/series, and the response will show movie statistics (year, actors, plot etc.). As if this wasn't enought the app allows you to make the "spotify this song" command which will open spotify in your browser ad play the song provided as a command line argument. Video demo can be seen below

TECHNOLOGIES USED:

  • Spotify API
  • Bands In Town API
  • OMDB API
  • Axios
  • Moment
